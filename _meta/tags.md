# Tags

Tag taxonomy in use. Hierarchical with `/` separator.

## Source types (literature notes)

- `#lit` — literature note
- `#lit/{field}` — by field, e.g. `#lit/medicine`, `#lit/methodology`

## Permanent note topics

- `#perm` — permanent note
- `#perm/{topic}` — by topic, e.g. `#perm/etiology`, `#perm/disease`, `#perm/mechanism`

## Structure

- `#moc` — map of content (index/hub note)

## Status

- `#fleeting` — transient, in daily notes
- `#todo` — needs work
- `#wip` — work in progress

## Conventions

- Prefer existing tags; only propose new ones if the note is clearly in a category not yet covered.
- Match the granularity of existing tags: don't use broad tags when specific ones exist.
- Permanent notes get atomic-concept tags. Literature notes get source-type tags.
- 1–4 tags per note. More than that is noise.
